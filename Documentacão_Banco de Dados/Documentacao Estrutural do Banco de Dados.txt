Documentação Estrutural do Banco de Dados: Sistema de Comércio Eletrônico ICommerce

1. Descrição Geral do Banco de Dados

* Nome do Banco de Dados: ICommerce
* Tipo de Servidor: Mecanismo de Banco de Dados (Microsoft SQL Server)
* Nome do Servidor: NATAN\SQLEXPRESS
* Autenticação: Autenticação do Windows
* Nome do Usuário: Natan/Natanael

Propósito:
O banco de dados "ICommerce" foi desenvolvido para suportar um sistema de comércio eletrônico. Ele armazena informações sobre produtos, compradores, vendedores, preferências de comprador, avaliações, carrinhos de compras e outras entidades relacionadas ao processo de compra e venda de produtos online.

Principais Funcionalidades:
* Gerenciamento de produtos e suas características, como descrição, preço, disponibilidade e estoque.
* Registro e gerenciamento de compradores, incluindo informações de contato e preferências.
* Cadastro e gestão de vendedores, juntamente com detalhes de suas lojas e produtos.
* Manutenção de avaliações de produtos feitas por compradores.
* Suporte para carrinhos de compras, permitindo que os compradores adicionem e removam itens durante o processo de compra.

Tecnologias Utilizadas:
* Microsoft SQL Server: Utilizado como o mecanismo de banco de dados para armazenamento e recuperação de dados.
* Microsoft SQL Server Management Studio: Utilizado para gerenciar o banco de dados, criar consultas SQL, gerenciar esquemas de banco de dados, etc.


2. Camadas de Aplicação

* Camada de Dados:
* A camada de dados da aplicação é responsável por realizar operações de leitura e escrita no banco de dados.
* A API utiliza consultas SQL para recuperar e modificar dados conforme necessário, seguindo as regras de negócio definidas.
* Camada de Negócios:
* Na camada de negócios, as regras de negócio da aplicação são implementadas.
* Isso inclui validações de dados, cálculos, processamento de lógica de negócios e qualquer outra operação que não seja diretamente relacionada à manipulação de dados brutos.
* Essas regras de negócio da API ainda vão ser defenidas.
* Camada de Apresentação:
* A camada de apresentação é representada por um aplicativo para celular.
* Este aplicativo oferece uma interface intuitiva e amigável para os usuários interagirem com o sistema "ICommerce".
* Através do aplicativo, os usuários podem visualizar produtos, adicionar itens ao carrinho, realizar compras, visualizar avaliações de produtos e interagir com outras funcionalidades oferecidas pelo sistema.
* Proteção via Token:
* Para garantir a segurança das interações com a API, é implementada uma camada de proteção via token.
* Cada solicitação feita à API deve incluir um token de autenticação válido, que é verificado e validado antes que a solicitação seja processada.
* Isso ajuda a proteger o banco de dados contra acesso não autorizado e garante que apenas usuários autenticados tenham permissão para realizar operações no sistema.
